【Q4OS】デフォルト起動OSの変更方法

exeインストーラーのデュアルブート環境設定

導入：発生している問題

Issue Q4OSをexeインストーラーで導入し、Windowsとのデュアルブート構成にした際、GRUBメニューで常に「Windows Boot Manager」がデフォルトに選択されてしまう。

通常、Linuxディストリビューションでは update-grub コマンドで設定を更新しますが、Q4OSのexeインストーラー環境ではこの操作だけでは設定が反映されません。

原因の解説

この問題の原因は、Q4OSのexeインストーラーが GRUB設定ファイル（grub.cfg）を標準とは異なるパスに生成していること にあります。

設定ファイル パス 備考
標準的なパス /boot/grub/grub.cfg update-grub が書き込む先
実際に使用されるパス /boot/efi/EFI/DebianBoot/grub.cfg UEFI起動時に参照される実体

実体ファイルに set default="1"（Windows側）がハードコードされており、標準の更新コマンドではこのファイルが書き換えられないため、設定が固定されてしまいます。

解決策：Q4OSをデフォルトにする

以下のステップで、正しいパスに対して設定を書き込みます。

1
GRUB基本設定ファイルの編集

/etc/default/grub を開き、GRUB_DEFAULT の値をQ4OSのメニュー名に変更します。

sudo nano /etc/default/grub

# 変更前
GRUB_DEFAULT=0

# 変更後 (メニュー名を正確に指定)
GRUB_DEFAULT="Q4OS Andromeda Desktop"
2
実体ファイルへの書き出し

update-grub の代わりに、出力先を明示的に指定して設定を生成します。

sudo grub-mkconfig -o /boot/efi/EFI/DebianBoot/grub.cfg
3
設定の確認

ファイル内に正しいデフォルト設定が書き込まれたか確認します。

sudo grep "set default" /boot/efi/EFI/DebianBoot/grub.cfg
4
再起動

システムを再起動し、Q4OSがデフォルトで選択されていることを確認してください。

元に戻す方法（Windowsをデフォルトにする）

再びWindowsを優先させたい場合は、同様の手順で設定値を変更します。

# 1. /etc/default/grub を編集 (メニュー名を正確に指定)
GRUB_DEFAULT="Windows Boot Manager (on /dev/sda2)"

# 2. 設定を実体ファイルに反映
sudo grub-mkconfig -o /boot/efi/EFI/DebianBoot/grub.cfg

# 3. 再起動

まとめと注意点

項目 重要ポイント
設定パス /boot/efi/EFI/DebianBoot/grub.cfg に grub-mkconfig で設定を反映させること
NG操作 単なる sudo update-grub では解決しない
指定方法 GRUB_DEFAULT にはメニュー名を正確に記述すること
（メニュー順が変わらなければ 0, 1, 2... などの番号指定でも動作します）
